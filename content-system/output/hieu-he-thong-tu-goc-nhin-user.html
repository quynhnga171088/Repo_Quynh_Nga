Hiểu hệ thống từ góc nhìn User

Không cần biết code — chỉ cần biết đặt câu hỏi đúng, bạn sẽ hiểu được hệ thống.

Quỳnh Nga BrSE Japan — Đồng Hành Cùng Bạn

Dành cho người mới · ~20 phút · 5 Phương pháp · Thực hành ngay

Mục tiêu
Sau bài này, bạn sẽ làm được gì?

Bài học này không đi theo hướng kỹ thuật. Thay vào đó, mình bắt đầu từ góc nhìn người dùng — cách tiếp cận dễ hiểu nhất cho người mới.

Bạn sẽ đạt được
  • Nhìn 1 website → hiểu nó dùng để làm gì
  • Tự xác định: User Type, Function List, Screen List
  • Hiểu được sự khác nhau giữa Site User và Site Admin
  • Có thể tự phân tích 1 hệ thống mới trong 1–2 ngày
Cấu trúc bài học: 4 Phương pháp và Cấu trúc cơ bản
  • Phương pháp 1 — Hiểu bằng trải nghiệm (User Thinking)
  • Phương pháp 2 — Hiểu bằng hành động (Function Thinking)
  • Phương pháp 3 — Hiểu bằng màn hình (Screen Thinking)
  • Phương pháp 4 — User Flow (Kết nối tất cả)
  • Cấu trúc cơ bản — Hiểu về Site và Platform
Phương pháp 1
Hiểu bằng trải nghiệm (User Thinking)

Đặt câu hỏi: "Website này để làm gì?"

1. Câu hỏi vàng (Golden Question)

Mỗi khi nhìn vào 1 website hay 1 hệ thống, hãy luôn bắt đầu bằng câu hỏi:

"User vào đây để làm gì?"
2. Xác định User Type

Một hệ thống thường có nhiều loại người dùng khác nhau. Bạn cần xác định: ai đang dùng hệ thống này, họ dùng với mục đích gì?

  • User — người dùng thường
  • Admin — người quản trị hệ thống
Ví dụ thực tế: Hệ thống Shopee

Shopee là một trang thương mại điện tử (EC site). Hãy thử phân tích:

  • Loại hệ thống: EC site (trang thương mại điện tử)
  • User là Customer → vào để mua hàng, tìm kiếm sản phẩm, đặt đơn
  • Admin → vào để quản lý sản phẩm, xử lý đơn hàng, xem báo cáo
Bạn không cần hiểu code hay kỹ thuật — chỉ cần mở website, dùng thử và nghĩ xem "Mình có thể làm gì trên này?"
Phương pháp 2
Hiểu bằng hành động (Function Thinking)

Bước tiếp theo: liệt kê tất cả những gì user có thể làm trên hệ thống.

1. Function là gì?
Function = 1 hành động mà user có thể làm trên hệ thống

Ví dụ các hành động: Đăng ký, Đăng nhập, Tìm kiếm, Mua hàng, Thanh toán...

2. Từ User Type → Viết Function List

Dựa trên kết quả Phương pháp 1, hãy liệt kê function cho từng loại user.

Output mẫu — Function list (Customer):
• Register (Đăng ký)
• Login (Đăng nhập)
• Search product (Tìm kiếm sản phẩm)
• Add to cart (Thêm vào giỏ hàng)
• Checkout (Thanh toán)
Bài tập Phương pháp 2

Chọn 1 website bất kỳ (Tiki, Lazada, YouTube...) → Liệt kê function list cho user.

Lưu ý: Viết theo từng loại user — đừng trộn lẫn function của User và Admin vào chung.
Phương pháp 3
Hiểu bằng màn hình (Screen Thinking)

Mỗi function thường đi kèm với 1 màn hình. Bước này giúp bạn "nhìn thấy" hệ thống.

1. Quy tắc đơn giản
1 Function → thường đi kèm với 1 Screen (màn hình)
2. Mapping: Function → Screen
FunctionScreen
LoginLogin screen
SearchSearch screen
Add to cartCart screen
CheckoutPayment screen
Bài tập Phương pháp 3

Convert function list ở Phương pháp 2 → screen list.

Output mẫu — Screen list:
• Login screen
• Home screen
• Product list screen
• Cart screen
• Payment screen
Phương pháp 4
User Flow — Kết nối tất cả

User flow là chuỗi các bước mà user thực hiện để hoàn thành 1 mục tiêu. Đây là xương sống của hệ thống.

Ví dụ: Flow mua hàng
Register → Login → Search → Add to cart → Checkout
Bài tập Phương pháp 4

Vẽ 1 user flow hoàn chỉnh cho hệ thống bạn đã phân tích từ Phương pháp 1 đến Phương pháp 3.

Hãy nghĩ theo hành trình thực tế: Nếu bạn là user, bạn sẽ làm gì đầu tiên? Rồi tiếp theo là gì? Cứ liệt kê theo thứ tự.
Site và Platform
Hiểu về Site và Platform

Ở phần này, mình muốn bạn nhìn “cấu trúc hệ thống” phân theo 2 góc độ khác nhau. Khi bạn ghép 2 góc độ này lại, bạn sẽ “ngộ ra” cách hệ thống được tổ chức.

🧩 01-Phân theo đối tượng sử dụng (User Type)

Hệ thống thường có ít nhất 2 nhóm người dùng, và mỗi nhóm có một “site” riêng.

  • Site User (User Site) — dành cho người dùng cuối (customer)
  • Site Admin (Admin Site) — dành cho người vận hành hệ thống
User Site thường có các hành động như: đăng ký/đăng nhập, sử dụng dịch vụ, xem/nhập dữ liệu.
Admin Site thường có: quản lý user, quản lý dữ liệu, cấu hình hệ thống.
Sơ đồ (theo User Type):

System
├── User Site (Customer)
└── Admin Site (Admin)
🧩 02-Phân theo nền tảng (Platform)

Cùng một hệ thống, nhưng có thể được triển khai trên nhiều nền tảng khác nhau.

  • Web System (Web App) — chạy trên trình duyệt (Chrome, Safari…). Không cần cài đặt, truy cập bằng URL.
  • Mobile App — chạy trên điện thoại (iOS/Android). Cần cài đặt, UI tối ưu cho mobile.
Sơ đồ (theo Platform):

System
├── Web (Browser)
└── Mobile App (iOS / Android)
Giải thích nhanh:
✔ User Site có thể có Web và/hoặc Mobile App (vì user cuối có thể dùng PC hoặc điện thoại).
✔ Admin Site có App không? Có thể có, nhưng đa số là chỉ có Web (vì admin cần màn hình lớn, nhiều dữ liệu, nhiều thao tác phức tạp -> làm việc trên web PC tiện hơn).
Bài tập

Chọn 1 hệ thống bạn quen thuộc, rồi trả lời 4 câu hỏi sau (đúng/sai cũng được, quan trọng là bạn biết cách nghĩ):

  • Hệ thống có User Site không? (Có) Nếu có: user vào để làm gì?
  • Hệ thống có Admin Site không? (Có) Nếu có: admin quản lý gì?
  • User Site đang chạy trên Web ? Có Mobile App không?
  • Admin Site có app không? Nếu không, lý do là gì?
Áp dụng thực tế
Template phân tích hệ thống

Dùng template dưới đây để phân tích bất kỳ hệ thống nào. Đây là kết quả tổng hợp của 5 Phương pháp bạn vừa học.

6 bước phân tích
  • 1. Overview — Hệ thống dùng để làm gì?
  • 2. User Types — Ai sử dụng? (Customer, Admin...)
  • 3. Function List — Mỗi loại user làm được gì?
  • 4. Screen List — Gồm những màn hình nào?
  • 5. User Flow — Luồng thao tác chính
  • 6. Site Structure — User Site, Admin Site, App
Thực hành: Chọn 1 website yêu thích → dùng template này để phân tích hoàn chỉnh. Khi làm xong, bạn đã có bản phân tích hệ thống đầu tiên của mình!

Chỉ với 5 bước đơn giản, bạn đã biết cách phân tích bất kỳ hệ thống nào — không cần biết code, không cần background kỹ thuật. Hãy luyện tập thường xuyên với các website xung quanh bạn.

5 ý chính cần nhớ
  • Luôn bắt đầu bằng câu hỏi: "User vào đây để làm gì?"
  • Phân tích theo 4 Phương pháp: User Type → Function → Screen → Flow
  • Site User dành cho khách hàng, Site Admin dành cho người vận hành
  • 1 Function thường đi kèm 1 Screen
  • User Flow = xương sống — kết nối tất cả lại với nhau
