Quy trình xây dựng một website

Trang này là “điểm quay lại” từ quiz. Bạn có thể thay nội dung bài học ở đây (hoặc gửi file bài học, mình sẽ nhúng vào).

← Trang chủ Làm quiz →

Tóm tắt nhanh (gợi ý)

  • Requirement: làm rõ nhu cầu, output thường là Spec/Wireframe/Flow nghiệp vụ.
  • Design: thiết kế UI/UX và kiến trúc/thiết kế hệ thống.
  • Coding: triển khai code theo thiết kế.
  • Testing: kiểm tra chất lượng, tìm lỗi.
  • Release/Deploy: đưa lên môi trường chạy thật.
  • Operation: vận hành, monitor, fix bug, update, backup.
Nếu bạn có file “Quy trình xây dựng một website.html” (tài liệu bài học đầy đủ), hãy đưa vào workspace; mình sẽ thay nội dung trang này bằng đúng tài liệu đó và giữ nguyên link qua lại với quiz.
