Mức 1 · Cơ bản
Câu 1
“Câu hỏi vàng” khi phân tích hệ thống từ góc User là gì?
  • A Hệ thống viết bằng ngôn ngữ gì?
  • B User vào đây để làm gì?
  • C Database có bao nhiêu bảng?
  • D Server đặt ở đâu?
Câu 2
Portal trong hệ thống được hiểu là gì?
  • A Cổng truy cập theo nhóm đối tượng (Admin portal, User portal…)
  • B Tên miền website
  • C Ngôn ngữ lập trình backend
  • D Loại database
Câu 3
Admin Portal thường được triển khai chủ yếu bằng hình thức nào?
  • A Chỉ Mobile App
  • B Chỉ SMS
  • C Web (trình duyệt PC), có thể kèm CMS
  • D Email
Câu 4
User Portal (ví dụ LMS) thường có những hình thức nào?
  • A Chỉ CMS nội bộ
  • B Web và/hoặc Mobile App
  • C Chỉ API document
  • D Chỉ Desktop cài trên server
Câu 5
Role trong hệ thống có nghĩa là gì?
  • A Tên file source code
  • B Loại server
  • C Ngôn ngữ giao tiếp với khách hàng
  • D Vai trò người dùng — quyền và công việc tương ứng
Câu 6
Function trong phân tích hệ thống là gì?
  • A Một hành động user có thể thực hiện trên hệ thống
  • B Tên công ty phát triển phần mềm
  • C Phiên bản hệ điều hành
  • D Hợp đồng lao động
Câu 7
Tích hợp Internal (nội bộ) nghĩa là gì?
  • A Kết nối với Facebook, Google
  • B Kết nối hệ thống cùng tổ chức (VD: HRM nội bộ)
  • C Không kết nối hệ thống nào khác
  • D Chỉ dùng Excel offline
Câu 8
Trong LMS, Primary user (người dùng chính) thường là ai?
  • A Admin hệ thống
  • B Giáo viên
  • C Học viên
  • D Kế toán công ty
Câu 9
CMS trong ngữ cảnh portal thường dùng để làm gì?
  • A Soạn và đăng nội dung — thường trong Admin portal
  • B Học viên làm bài quiz
  • C Thanh toán học phí
  • D Gửi email marketing
Câu 10
“Điểm cần xác nhận” nên dùng khi nào?
  • A Sau khi deploy production 1 năm
  • B Khi viết code xong hết
  • C Chỉ khi nghỉ việc
  • D Khi trình bày lại hiểu biết với khách hàng / team

Câu 11
Portal và Web / Mobile App / CMS khác nhau thế nào?
  • A Là một — chỉ khác tên gọi
  • B Portal thay thế hoàn toàn cho Function
  • C Portal = cổng theo đối tượng; Web/App/CMS = hình thức triển khai
  • D CMS luôn là portal riêng cho học viên
Câu 12
Khi lập Function list, quy tắc vàng là gì?
  • A Gộp tất cả role vào một bảng chung
  • B Viết riêng theo từng Role — không trộn Admin với Học viên
  • C Chỉ liệt kê tên màn hình, không cần hành động
  • D Copy từ tài liệu đối thủ, không cần xác nhận
Câu 13
Udemy / Coursera tích hợp với LMS thường được phân loại là gì?
  • A External — hệ thống bên thứ ba
  • B Internal — cùng công ty
  • C Không phải tích hợp
  • D Primary user
Câu 14
Thuật ngữ tiếng Nhật 受講 (juko) trong LMS thường nghĩa là gì?
  • A Chấm điểm bài tập
  • B Xóa tài khoản
  • C Đăng ký / tham gia học (Enrollment)
  • D Cấu hình server
Câu 15
修了 (shuryo) trong ngữ cảnh đào tạo thường có nghĩa là gì?
  • A Bắt đầu khóa học
  • B Hoàn thành khóa học / đủ điều kiện kết thúc
  • C Đăng nhập lần đầu
  • D Upload video
Câu 16
Glossary nên được lập như thế nào khi làm dự án Nhật?
  • A Chỉ dịch Google một lần, không cần cập nhật
  • B Chỉ ghi tiếng Việt, bỏ qua thuật ngữ gốc
  • C Không cần vì đã biết tiếng Nhật
  • D Từ tài liệu/màn hình/họp; có thể dùng AI nhưng phải đối chiếu
Câu 17
Quan hệ Function và Screen thường là gì?
  • A Thường 1 Function chính → 1 Screen chính
  • B Screen không liên quan Function
  • C Một Screen luôn có 10 Function
  • D Function chỉ dành cho dev, không cho BrSE

Thứ tự phân tích hệ thống đúng theo bài học là gì?
  • A Function → Role → Portal → Mục đích
  • B Tích hợp → Glossary → Function
  • C Khái quát → Role → Function/Screen → User Flow & Detail → Glossary

Câu 19
Teacher Leader trong LMS khác Teacher thường ở điểm nào?
  • A Chỉ học viên mới có Leader
  • B Thêm quyền duyệt bài / giám sát nhóm giáo viên
  • C Không đăng nhập được
  • D Chỉ dùng Mobile App
Câu 20
VNPay tích hợp LMS để làm gì? Mục đích tích hợp là gì?

  • B Đồng bộ danh sách nhân sự nội bộ
  • C Soạn bài giảng bằng video
  • D Dịch tài liệu sang tiếng Nhật
